## 2026-05-19 v2.0 每日灵活交易系统

### 设计理念
- 不再月度锁定，改为每日信号驱动的灵活买卖
- 板块优先：先选热点板块（动量+广度+量能），再选个股
- 个股多因子：RSI + MACD + 布林带 + 量比 + 动量

### 实现文件
| 文件 | 功能 |
|------|------|
| v2/sector_classifier.py | 板块分类（82个行业/板块） |
| v2/engine.py | 每日信号回测引擎 |
| v2/report.py | HTML对比报告（v2.0 vs v1.5） |

### 回测结果 (2025.04 ~ 2026.05)
- 初始资金: 100,000 → 最终市值: 42,191
- **总收益: -57.81%**（大幅亏损）
- 交易: 260买 / 260卖 | 胜率: 40.0%
- 止损 -8% | 止盈 +20% | 最长持有25天

### 问题分析
1. **过度交易**：271个交易日260笔买卖，几乎每天都在换仓，手续费累积严重（0.15%×2×260≈7800元）
2. **胜率过低**：40%胜率配合不对称的止损止盈比例（8% vs 20%），理论上需要>71%的胜率才能不亏
3. **买入阈值太低**：score > 0.02 即买入，筛选不够严格
4. **板块轮动太快**：每5天重评板块，可能频繁追热点却总是追晚了
5. **止损频繁触发**：-8%止损在震荡市中容易被扫

### 改进方向（v2.1）
- [ ] 提高买入阈值（0.05以上）
- [ ] 降低交易频率（避免频繁换仓）
- [ ] 调整止损止盈比例（如 -5% / +15% 更快止盈）
- [ ] 加入大盘趋势过滤（熊市不买）
- [ ] 引入板块持续性判断（连续热点板块优先）
